## SKYNET — Admin Panel & User Management — Feature Request

### Bug Fixes Required

**1. Admin user list is empty**
When logged in as admin, the Users section shows no data. Fix this so all registered users are fetched and displayed in a table — showing username, email, role, account status, join date, and total PPTs generated.

**2. User-generated PPTs not visible to admin**
Admin cannot see PPTs generated by other users. Fix this so admin has a global view of all generations across all accounts, with a label showing who created each one — format: `Generated by: username`.

---

### New Features to Build

**3. Admin can create user accounts**
Add a "Create Account" button in the admin panel that opens a form with fields for username, email, password, and role (User or Admin). On submit, the account is created and the user receives a welcome email or can log in immediately depending on role.

**4. Admin account creation requires master approval**
When someone registers as Admin role, the request must NOT activate immediately. Instead it goes into a pending state and sends a notification to the master account (super admin). The master account sees a pending approvals queue and must click Approve or Reject before that admin account can log in.

**5. Login page — User vs Admin tab selector**
On the login page, add a toggle or tab switcher between `Employee / User` and `Admin` modes exactly like the iamneo Timesheet login. When Admin tab is selected, show a different credential hint and route them to the admin dashboard on successful auth.

**6. "Generated by" label on all PPT cards**
Every PPT card in the admin global view must show a label — `Generated by: username` — so admin can identify ownership at a glance. Style it as a small monospace badge consistent with the SKYNET cyberpunk theme.

---

### Full Flow Summary

```
User registers → standard access → can log in immediately

Admin registers → status: PENDING → notification sent to master
                → master approves → status: ACTIVE → can now log in
                → master rejects  → status: REJECTED → login blocked

Login page → Employee tab → routes to user dashboard
          → Admin tab    → routes to admin dashboard

Admin dashboard → sees all users list
               → sees all PPTs with "Generated by: username" label
               → can create new accounts (user or admin)
               → can approve/reject pending admin requests
```

### API Endpoints Needed

```
GET    /admin/users                → list all users
GET    /admin/generations          → list all PPTs across all users
POST   /admin/users/create         → create a new account
POST   /admin/approve/:userId      → master approves pending admin
POST   /admin/reject/:userId       → master rejects pending admin
GET    /admin/pending              → list all pending admin requests
```

---

### Notes for Antigravity

- The master account is a hardcoded super-admin — it cannot be created through the UI, only seeded in the database directly
- Admin approval notifications can be a simple in-app badge count on the master account header — no email needed for v1
- The `Generated by: username` label should appear on every PPT card in admin view, styled as a small cyan monospace tag in the SKYNET theme
- All new UI components must match the existing SKYNET cyberpunk theme — dark background, cyan borders, Orbitron font for headings, Share Tech Mono for labels
